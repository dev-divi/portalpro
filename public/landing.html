🔒 Your client's data never touches a server.

PortalPro uses AES-GCM encryption via the browser's native Web Crypto API. The password derives an encryption key with PBKDF2. The encrypted payload is base64-encoded and embedded in the URL hash. No data is transmitted to any server, ever. This isn't a marketing claim — it's the architecture.

Questions

Is client data really secure?

Yes. PortalPro encrypts all project data in your browser before it goes anywhere. The encryption uses AES-GCM with PBKDF2 key derivation — the same standard used by password managers. No server ever sees the unencrypted data.

Does my client need an account?

No. You send them a link. They enter the password you gave them. The portal decrypts and loads. No signup, no login, no account creation.

What happens if I lose the password?

The password is the only way to decrypt the portal. There is no "reset" because there is no server storing your data. Keep a copy of your password in a password manager.

Can I use my own domain?

Yes. PortalPro is a static app you can host on your own domain via Netlify, Vercel, or any static host. Your client will see your URL, not ours.

How is this different from Notion or Monday.com?

Notion and Monday are general-purpose project tools with monthly seats, databases, and admin overhead. PortalPro is purpose-built for client-facing dashboards: encrypted, branded, no-login, shareable as a link. It's not a project management tool — it's a client experience tool.
